# Tessera Partners Website

Marketing site for [tesserapartners.co](https://tesserapartners.co) — built with vanilla HTML/CSS/JS, CMS-powered by TinaCMS, deployed on Netlify.

---

## How it works

All site copy lives in [`content/site.json`](content/site.json). A lightweight Node.js build script reads that file and renders it into static HTML. TinaCMS provides a visual editor at `/admin` that commits changes directly back to GitHub, triggering a Netlify rebuild.

```
Edit in TinaCMS → commit to GitHub → Netlify rebuilds → live site updated
```

---

## Local development

**Prerequisites:** Node.js 18+

```bash
npm install
```

### Build the static site only (no CMS)

```bash
node build.js
# Output → public/index.html, public/styles.css, public/main.js
```

Open `public/index.html` in a browser to preview.

### Run with TinaCMS dev server

```bash
npm run dev
```

This starts TinaCMS in local mode (no cloud auth required) and rebuilds the site on content changes. Visit `http://localhost:3000` for the site and `http://localhost:3000/admin` for the CMS editor.

## Deployment to Netlify

### 1. Push to GitHub

Create a new GitHub repository and push this project:

```bash
git init
git add .
git commit -m "Initial site"
git remote add origin https://github.com/YOUR_USERNAME/tessera-partners-website.git
git push -u origin main
```

### 2. Connect Tina Cloud

1. Go to [tina.io](https://tina.io) and sign in (or create a free account).
2. Create a new project and connect your GitHub repository.
3. Tina will provide two environment variables:
   - `TINA_CLIENT_ID`
   - `TINA_TOKEN`

Keep these — you'll need them for Netlify.

### 3. Deploy on Netlify

1. Go to [netlify.com](https://netlify.com) → **Add new site → Import an existing project**.
2. Select your GitHub repository.
3. Build settings are auto-detected from `netlify.toml`:
   - **Build command:** `npm run build`
   - **Publish directory:** `public`
4. Before deploying, add the Tina Cloud environment variables:
   - Go to **Site settings → Environment variables**
   - Add `TINA_CLIENT_ID` and `TINA_TOKEN` from step 2

5. Click **Deploy site**.

### 4. Configure custom domain

1. In Netlify: **Site settings → Domain management → Add custom domain**
2. Enter `tesserapartners.co`
3. Follow Netlify's DNS instructions (update nameservers or add a CNAME/A record at your registrar)
4. Netlify provisions free SSL automatically once DNS propagates (usually within minutes)

## Making content edits via the CMS

Once deployed, the editorial interface is available at:

```
https://tesserapartners.co/admin
```

1. Visit `/admin` and sign in with your Tina Cloud account.
2. You'll see a single document: **Site Content**.
3. Click to open it — every section of the site is editable: Nav, Hero, Credibility Strip, Who I Work With, Services, PMOS, Approach, About, Contact, and Footer.
4. Make your changes and click **Save**.
5. TinaCMS commits the updated `content/site.json` to GitHub.
6. Netlify detects the new commit and rebuilds the site automatically (typically 30–60 seconds).

## Project structure

```
├── content/
│   └── site.json        # All site copy — edit this via TinaCMS or directly
├── tina/
│   └── config.js        # TinaCMS schema (all editable fields registered here)
├── src/
│   ├── template.js      # HTML template function
│   ├── styles.css       # All styles (mobile-first, no frameworks)
│   └── main.js          # Client-side JS (scroll, hamburger nav)
├── build.js             # Build script: content/site.json → public/
├── netlify.toml         # Netlify build config
└── package.json
```

**Build output** (`public/` — gitignored):
```
public/
├── index.html
├── styles.css
├── main.js
└── admin/               # Generated by TinaCMS build
```

## Environment variables reference

| Variable | Where to get it | Required for |
|---|---|---|
| `TINA_CLIENT_ID` | Tina Cloud project settings | Netlify build, local dev with cloud |
| `TINA_TOKEN` | Tina Cloud project settings | Netlify build, local dev with cloud |

Local dev without cloud auth uses `TINA_PUBLIC_IS_LOCAL=true` (set automatically by `npm run dev`).
